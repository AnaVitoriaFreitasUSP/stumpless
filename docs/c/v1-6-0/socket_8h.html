Detailed Description

Socket targets allow logs to be sent to a Unix domain socket.

The most common use for socket targets is to send logs to the /dev/log socket or the /var/run/syslog where the syslog daemon is listening.

Opens a socket target.

Parameters
nameThe name of the socket target, and the socket to send logs to.
local_socketThe name of the Unix domain socket to connect from. This can be set to NULL if you'd like to use a name generated by the library. Generated names begin with 'stumpless' and are followed by six random characters.
optionsThe options to use for the logging target. This is a bit-wise or of one or more STUMPLESS_OPTION values.
default_facilityThe facility code to use for entries that do not have a facility value set on them. This should be a STUMPLESS_FACILITY value.
Returns
The opened target if no error is encountered. In the event of an error, NULL is returned and an error code is set appropriately.
